# CLAUDE.md

## 核心约定

本目录是个人生活管理空间，不是代码项目。所有交互使用中文。

## 记账系统

当用户提及金额、消费、收入、花销等与财务相关的信息时，按以下规则处理。

### 时间规则

所有记账日期以系统提供的 `currentDate` 为准（互联网时间），不使用用户本地系统时间，因为用户本地时间可能被修改而不准确。

### 记账规则

用户可以用自然语言描述收支，不用严格格式。你需要做：

1. 识别这是支出还是收入
2. 匹配到预设类别
3. 追加记录到 `记账本.md` 当前月份表格
4. 自动更新文件顶部「当前余额」
5. 追加后简要确认：「已记：午餐 35 → 餐饮 | 余额 28,465」

### 支出类别（可扩展）

餐饮、交通、购物、娱乐、居家、医疗、人情、还款、其他

### 收入类别

收入（工资、红包等）、收回借款

### 多账户余额

用户不必手动算总余额。只需告诉各账户有多少钱，自动加总。

**识别规则**：当用户说「账户名 + 金额」（如「微信 5000」「银行卡余额 20000」「支付宝还有 800」），且上下文明显不是消费时，识别为账户余额更新：

1. 更新 `记账本.md` 顶部账户余额表（新增账户或更新已有账户）
2. 自动重算合计 = 所有账户余额之和
3. 追加一条「余额校准」记录到当前月份表（标注哪个账户）
4. 确认：「已更新：微信 5,000 | 合计 25,000」

**常见账户名自动识别**：微信、支付宝、银行卡、工行、建行、招行、现金、储蓄卡、信用卡、零钱通、余额宝

用户说新名字时自动加入账户列表。基金、股票归入投资板块，不在此处。

**注意**：若用户明确在描述消费（「微信支付了 35 午餐」），这仍是支出，不是调整账户余额。

### 投资跟踪

基金、股票等投资品，记录投入本金、当前市值、已卖出金额，自动算盈亏。

**登记/更新**：用户说投资信息时：

- 「基金 投入 25000 现在市值 30000」
- 「股票 本金 18000 现值 20000」
- 「基金现在涨到 32000」

1. 更新 `记账本.md` 投资表。**投入**列是累计投入本金，后续不随市值波动改变（除非用户追加投入）
2. 总盈亏 = 当前市值 + 已卖出 - 投入
3. 确认：「已更新：基金 投入 25,000 市值 30,000 ｜ 盈亏 +5,000」

**卖出**：用户说卖出了投资品时：

- 「基金卖了 5000」
- 「股票卖了 8000 还剩 12000」

1. 已卖出 += 卖出金额，当前市值 -= 卖出金额（若用户报了剩余市值则以用户报的为准）
2. 将卖出金额加入现金账户余额（用户指定转入哪个账户，未指定则默认问一句或加到银行卡）
3. 若全部卖完（当前市值归零），保留记录但标记
4. 确认：「已记录：基金卖出 5,000 ｜ 已转入余额 ｜ 剩余市值 25,000」

**卖出金额自动计入余额**：卖出后现金账户增加，无需用户再单独更新账户余额。

**只报市值**：若用户只说「基金 30000」没提投入，只更新市值列。若该投资品不存在则询问投入。

**更新频率**：用户偶尔报最新市值即可，不需要每次变动都记。

### 借出管理

借给他人的钱，记录借款人、总额、已还、待还。

**登记借出**：

- 「借给张三 5000」
- 「借了 2000 给李四 上个月的事」
- 「张三借了 3000 已经还了 1000」

1. 更新 `记账本.md` 借出表（新增借款人或更新已有）
2. 确认：「已登记：张三 借出 5,000 ｜ 待还 5,000」

**记录还款**：用户说「XX 还了 Y」时：

- 「张三还了 1000」
- 「李四还清了」

1. 更新已还和待还，重新计算合计。若「还清了」，待还归零
2. 记一条收入（类别「还款收入」或「收回借款」）到当月收支表
3. 确认：「已记：张三还 1,000 ｜ 剩余待还 4,000」

**借出 vs 消费**：借出不是支出，是资产转移。记录借出不记消费支出，只在借出表中登记并更新总资产公式。

### 负债与分期

用户有分期还款、花呗、信用卡等负债。规则：

**登记负债**：用户描述分期信息时，登记到「负债与分期」表。

- 「手机分期总额 3000 每月 500 还剩 6 期 每月 15 号还」
- 「花呗这个月要还 1200」
- 「信用卡账单 2000 每月 25 号还」

花呗/信用卡这种金额每月变化的，类型标为「循环」，剩余总额标为「本月待还」，剩余期数标「-」。

**记录还款**：用户说「还了 XX」时：

1. 记一条支出（类别「还款」或「分期」）
2. 更新负债表中对应项的剩余总额和剩余期数
3. 若剩余期数归零，标记为已还清，保留在表中但不从合计扣
4. 确认：「已记还款：手机分期 500 | 剩余 5 期 / 2,500」

**还款日提醒**：用户问「还款」或「负债」时，列出所有负债、每月还款额、还款日、剩余期数。若当天日期距某还款日在 3 天内，主动提醒。

### 余额校准

用户说「总余额 X」（不指定具体账户）时，在表格中追加一条类型为「余额校准」的记录，更新合计。

### 查询命令

| 用户说 | 做什么 |
|--------|--------|
| `查账` | 本月汇总：总收入、总支出、各类别金额和占比、当前余额 |
| `查账 X月` | 指定月份汇总 |
| `余额` | 现金账户 + 投资 + 借出 + 负债 + 总资产 全览 |
| `投资` | 投资品明细 + 总盈亏 |
| `借出` | 借款明细：谁、借了多少、已还、待还 |
| `负债` / `还款` | 列出所有负债：名称、每月还款、剩余期数、还款日，近 3 天内到期的加提醒 |
| `近N天` | 最近 N 天消费流水 |
| `删最后一条` | 删除最近追加的那条记录，重新计算余额 |

### 特殊处理

- 若用户说「不对，那是 XX 类」，修改最近那条记录的类别
- 若用户说「改余额 X」，等同于余额校准
- 模糊类别（如「买东西」）默认为「购物」，除非上下文暗示其他
- 用户说「加个类别 XXX」时，将该类别加入可用列表并确认

### 数据文件

- `记账本.md`：账户余额 → 投资 → 借出 → 负债与分期 → 每月收支表，五层结构，顶部有总资产公式
- 账户表格式：`| 账户 | 余额 |`
- 投资表格式：`| 名称 | 投入 | 当前市值 | 已卖出 | 总盈亏 |`，总盈亏 = 当前市值 + 已卖出 - 投入
- 借出表格式：`| 借款人 | 借出总额 | 已还 | 待还 | 备注 |`
- 负债表格式：`| 名称 | 类型 | 剩余总额 | 每月还款 | 剩余期数 | 还款日 | 备注 |`
- 收支表格式：`| 日期 | 类型 | 类别 | 金额 | 备注 |`
- 总资产 = 现金合计 + 投资市值合计 + 借出待还合计 — 负债剩余总额
